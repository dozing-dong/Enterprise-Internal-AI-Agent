# Knowledge Chat — Frontend

A minimal React + TypeScript chat UI for the RAG backend, styled with
Apple "Liquid Glass" aesthetics (frosted blur, animated gradient mesh,
translucent panels).

## Stack

- Vite + React 18 + TypeScript
- Tailwind CSS 3 + shadcn-style primitives on top of Radix UI
- Zustand for state, lucide-react for icons
- POST + `ReadableStream` based SSE client (works with the `/chat/stream`
  FastAPI endpoint that takes a JSON body)

## Getting started

Make sure the backend is running first — the frontend dev server proxies
`/chat`, `/chat/stream`, `/sessions`, `/history`, and `/health` to it:

```bash
# From the repo root, in a separate terminal:
python run_api.py
# Backend now listening on http://127.0.0.1:8000
```

Then in `frontend/`:

```bash
npm install
npm run dev
```

The app opens at <http://localhost:5173>.

## How it talks to the backend

| UI action                | HTTP call                              |
| ------------------------ | -------------------------------------- |
| App boot                 | `GET /sessions`                        |
| Click **New chat**       | `POST /sessions` (server-generated ID) |
| Pick a session           | `GET /history/{session_id}`            |
| Send a message           | `POST /chat/stream` (SSE)              |
| Rename / delete a chat   | `PATCH` / `DELETE /sessions/{id}`      |

The streaming response uses these SSE event names:

- `progress` — stage indicator (`rewriting`, `retrieving`, `generating`,
  `titling`)
- `sources` — retrieval results, shown via the "N sources" popover
- `token` — incremental answer text appended to the assistant bubble
- `done` — final payload (`session_id`, generated `title`, `full_answer`)
- `error` — fatal error, surfaced as a dismissible banner

## Session conventions

- `session_id` is **always** generated by the backend
  (`uuid.uuid4().hex`, 32 chars). The frontend never invents one.
- The active `session_id` is persisted in `localStorage`
  (`rag-chat:active-session-id`) so a refresh keeps you in the same
  chat.
- The first turn of a fresh chat triggers backend LLM-based titling;
  the sidebar refreshes automatically once the stream finishes.

## Notes

- All UI copy is English. No localization layer is wired up.
- `node_modules/` and `dist/` are ignored both at the repo root and
  in `frontend/.gitignore` so they never get committed.
- Backend CORS only whitelists `http://localhost:5173` and
  `http://127.0.0.1:5173` for development. Update
  `backend/api/app.py` before deploying behind a different origin.
